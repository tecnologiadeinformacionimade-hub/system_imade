Infraestructura de Red
Requisitos de conectividad LAN y puertos TCP

La comunicación entre el dispositivo biométrico y el servidor ZKBioTime PRO se realiza mediante sockets TCP persistentes. Para que funcione de forma estable y sin pérdida de eventos, ambos equipos deben cumplir los siguientes requisitos:

•
El dispositivo biométrico y el servidor deben estar en la misma red local (LAN). No se admiten comunicaciones a través de NAT doble ni segmentos separados sin enrutamiento explícito.
•
La IP del biométrico debe ser estática: 192.168.1.152. Configúrela directamente en el firmware para evitar reasignaciones por DHCP.
•
El servidor debe tener abierto el puerto TCP 4370 para el protocolo PUSH y el puerto de la interfaz web.
Verificación de Conectividad ICMP

Antes de parametrizar el software, ejecute un ping continuo desde el servidor hacia el biométrico para confirmar respuesta estable sin pérdida de paquetes:

CMD C:\> ping 192.168.1.152 -t

Una latencia menor a 5ms confirma que la capa física y de enlace están correctas. Si hay pérdida mayor al 1%, revise el switch, el cable o el firewall antes de continuar.

Validación de Salida a Internet
CMD C:\> ping 8.8.8.8 -t

Necesario para que el servidor resuelva nombres de dominio usados por el cliente SMTP de Office 365.

Servidor ZKBioTime IP local del servidor Backend, PostgreSQL, Task Scheduler

Arquitectura del Script BAT
generar_y_enviar.bat — Orquestador principal del pipeline

Este script de Windows Batch actúa como orquestador principal del pipeline de reportes. Su función es establecer el contexto de ejecución, consultar PostgreSQL y delegar el procesamiento a PowerShell.

Bloque 1 — Conexión a PostgreSQL

Define las variables de entorno para que psql.exe se conecte al motor embebido. PostgreSQL corre localmente en el puerto no estándar 7496 para evitar conflictos con otras instancias.

BAT set "PSQL=C:\BioTime\pgsql\bin\psql.exe" set "HOST=127.0.0.1" set "PORT=7496" set "USER=postgres" set "DB=biotime" set "PGPASSWORD=SOLUCION_DB_PASS_AQUI"
Bloque 2 — Cálculo del Número de Semana

Invoca PowerShell inline para calcular el número ISO de la semana anterior. Si la semana actual es la 1, devuelve la 52 para cerrar el ciclo anual correctamente. Usa el estándar FirstFourDayWeek alineado con ISO 8601.

BAT for /f %%i in ('powershell -NoProfile -Command "$w = [CultureInfo]::..."') do set SEMANA_REPORTE=%%i
Bloque 3 — Consulta SQL y Extracción CSV

Núcleo del script. Ejecuta una query compleja sobre la base de datos que combina tres técnicas:

•
generate_series(): Genera el calendario completo lunes a domingo. Garantiza que todos los días aparezcan aunque no haya marcajes.
•
CROSS JOIN: Combina cada empleado activo con cada día del calendario. Los días sin marcaje quedan con hora nula.
•
LEFT JOIN a iclock_transaction: Adjunta los marcajes reales con conversión de zona horaria AT TIME ZONE 'America/Guatemala'.
SQL SELECT e.emp_code, nombre, departamento, fecha, hora FROM personnel_employee e CROSS JOIN generate_series(...) d_serie LEFT JOIN iclock_transaction t ON e.id = t.emp_id AND (t.punch_time AT TIME ZONE 'America/Guatemala')::date = d_serie.fecha WHERE e.enable_att IS TRUE ORDER BY fecha, emp_code, hora
Bloque 4 — Delegación a PowerShell

Si el CSV fue generado correctamente, el BAT llama al motor PowerShell pasando la ruta del archivo y el número de semana como argumentos. Si el archivo no existe (fallo en la query), imprime un error y termina sin enviar nada.

8
Motor PowerShell
ejecutador.ps1 — Procesamiento, Excel y envío por correo

Recibe el CSV crudo, aplica la lógica de negocio para calcular horas trabajadas, genera el reporte Excel formateado y lo despacha por correo electrónico.

Fase 1 — Validaciones Iniciales

Verifica que el archivo CSV exista y no esté vacío. Si alguna condición falla, el script termina con código de error sin generar archivos parciales ni enviar correos incompletos.

Fase 2 — Algoritmo de Pivote de Marcajes

El CSV contiene una fila por marcaje. El script agrupa por empleado_id + fecha y aplica el siguiente algoritmo:

Marcajes del díaAsignaciónEscenario
1 marcajeSolo EntradaOlvidó marcar la salida
2 marcajesEntrada + Salida FinalJornada sin almuerzo registrado
3 marcajesEntrada + Salida Almuerzo + Salida FinalAlmuerzo con una sola marca
4+ marcajesEntrada + Salida Almuerzo + Entrada Almuerzo + Salida FinalJornada completa

Con los pivotes asignados calcula: Tiempo de almuerzo, Jornada bruta, Jornada neta y Horas extras (minutos trabajados después de las 17:00).

Fase 3 — Generación del Excel con COM

Utiliza objetos COM nativos de Windows (Excel.Application) para crear el archivo .xlsx con formato profesional, sin módulos externos adicionales.

•
Encabezados con fondo oscuro y texto blanco en negrita.
•
Filas con colores alternados por fecha para facilitar la lectura.
•
Entradas tardías (después de las 7:10 a.m.) resaltadas en rojo.
•
Marcajes incompletos también en rojo negrita.
•
Primera fila fija (freeze pane) y autoajuste de columnas.

generar_y_enviar.bat — Orquestador principal

Este script de Windows Batch es el punto de entrada de todo el proceso. Se encarga de configurar la conexión a PostgreSQL, calcular la semana de reporte, ejecutar la consulta SQL para extraer los marcajes y delegar el procesamiento al motor PowerShell. Es invocado automáticamente por el Programador de Tareas cada lunes a las 06:30 a.m.

Estructura del script: Bloque 1 configura la conexión DB · Bloque 2 calcula el número de semana ISO · Bloque 3 ejecuta la query SQL y genera el CSV · Bloque 4 llama a PowerShell si el CSV existe, o imprime error si falló.
